2. Créer une image sur la base d’un Dockerfile

Conteneurisation d’un script

  • À l’aide du langage de programmation de votre choix, créer une image check-chapter qui contient un script :
    • faisant une requête GET sur l’URL : http://www.volonte-d.com/
    • affichant un message indiquant si la chaîne de caractères “Chapitre 1089” est présente sur la page et un autre message dans le cas contraire
  • Créer et exécuter un conteneur sur la base de cette image et vérifier qu’il fonctionne
  • Créer un dossier check-chapter à la racine du repository et y ajouter tout le code nécessaire pour créer l’image dans un sous-dossier script. Add / Commit / Push

Conteneurisation d’un serveur HTTP

  • Reproduire la démarche précédente, mais cette fois ci le message doit être accessible via un navigateur en accédant à localhost:8000
  • Ajouter le code dans le dossier check-chapter/webserver

3. Déployer une APP localement

Création de l’app

  • Créer un dossier app dans le dossier check-chapter La structure doit être la suivante : 
    app
├── Dockerfile
└── app
    ├── app.py
    └── requirements.txt
  • Fichierapp.py
    from flask import Flask

app = Flask(__name__)


@app.route("/")
def hello():
    return "Hello world"


if __name__ == "__main__":
    app.run(host="0.0.0.0", port=80)
  • Fichier requirements.txt :
    flask
  • Dockerfile
    FROM python:3.9-slim

WORKDIR /opt/demo/

COPY /app .

RUN pip install -r requirements.txt

ENTRYPOINT python app.py
  • Structure du dossier : 
    # docker-1/simple-app
.
└── app
		├── Dockerfile
    ├── app.py
    └── requirements.txt
  • Bâtir l’image et lancer l’app avec :
    docker build -t simple-app:latest .
docker run -d -p 80:80  simple-app:latest

Questions

  • Expliquer le rôle de l’option -t de la commande build
  • Expliquer le rôle de l’option -d de la commande run
  • Expliquer les commandes utilisées dans le Dockerfile
  • Ajouter une route /check-chapter à l’application qui affiche le message tel que décrit dans l’exercice 1
  • Relancer l’app pour qu’elle soit accessible via le port 8001
  • Add / Commit / Push pour sauvegarder le contenu du dossier check-chapter/app contenant l’app
