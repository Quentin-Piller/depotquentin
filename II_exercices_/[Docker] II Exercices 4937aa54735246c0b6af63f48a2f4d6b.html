C - Named Volumes

  • Pour inspecter les volumes existants et supprimer les non utilisés : 
    # Lister les volumes
docker volume ls 

# Supprimer les volumes non utilisés
docker volume prune

# Supprimer un volume sur la base de son Id
docker volume rm [volume]
  • Créer un volume avec : 
    # si on ne spécifie par le nom, alors un nom aléatoire est attribué
docker volume create vol

# On peut inspecter le volume avec
docker volume inspect vol
  • Répéter l’opération précédente d’écriture sans un fichier avec :

    (Ici, on utilise un bind-mount, pour comparer avec le volume)

    docker run -v $(pwd):/var/opt/project bash:latest \
  bash -c "ls /var/opt/project"
# file.txt
  • Cette fois-ci, utiliser : 
    docker run -v vol:/var/opt/project bash:latest \
  bash -c "echo Bonjour encore > /var/opt/project/file2.txt"
  • Vérifier que le fichier existe encore : 
    docker run -v vol:/var/opt/project bash -c "ls /var/opt/project"
  • Syntaxe alternative : option mount
    docker run --mount 'source=vol,dst=/var/opt/project' \
	bash -c "ls /var/opt/project"

6. Plusieurs conteneurs, un seul volume

  • Reprendre le code de l’app check-chapter/app de l’exercice 1, et écrire un script dans le langage de votre choix qui permettent de :
    • lancer 10 conteneurs servant tous l’app sur un port différent.
    • Il ne faut pas lancer 10 fois manuellement mais produire une procédure automatisée - bash est un bon choix, mais tout langage que vous maîtrisez mieux est un meilleur choix
  • Mettre tout votre code dans un dossier multiple-replicas en racine du repo
  • Numéroter les conteneurs de 1 à 10 et adapter le code pour que les 10 conteneurs puissent :
    • écrire une ligne de log lorsque une requête est faite sur la route d’url racine / (hello-world)
    • d’abord dans le même fichier dans le même bind-mount
      • Le bind-mount doit se trouver dans le repository
    • puis dans le même fichier dans un même volume que vous créerez
    • en précisant le numéro du conteneur dans chaque ligne de log
  • Vérifier que le dossier contient bien les lignes de logs par conteneur
